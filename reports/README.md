## Overview

This repository is a demonstration project for the FusionCode squad platform. It showcases how a team of AI agents can collaborate to:

- Extract design specifications from Figma files via MCP (Model Context Protocol)
- Generate pixel-perfect HTML/CSS or React+TypeScript+Tailwind implementations from those designs
- Perform automated code review, accessibility audits, and deploy checks
- Manage GitHub issues, commits, and pull requests autonomously

The primary deliverable has been a **Trello-style Kanban Board** built with React 18, TypeScript, Tailwind CSS, Zustand for state management, and `@dnd-kit` for drag-and-drop.

## AI Agent Skills

The `.fusion-plugin/skills/` directory contains reusable agent skill definitions. Each skill is a markdown file that instructs an AI agent how to perform a specific task.

| Skill | Description |
|---|---|
| `analyze-codebase` | Analyzes project structure, dependencies, and architecture |
| `close-issue` | Closes a GitHub issue |
| `code-review` | Reviews code for quality, security, and best practices |
| `create-issue` | Creates a new GitHub issue |
| `db-migration` | Creates and runs database migrations (TypeORM, Prisma, Knex, Sequelize) |
| `deploy-check` | Runs a full pre-deploy checklist (build, lint, tests, types, deps, env) |
| `figma-extract` | Extracts design data (nodes, images, styles) from a Figma file via MCP |
| `git-commit` | Stages and commits changes with a semantic commit message |
| `git-push-pr` | Pushes the current branch and opens a Pull Request on GitHub/GitLab |
| `jira-sync` | Syncs status and comments with Jira issues via MCP |
| `mobile-news` | Fetches recent mobile tech news and saves to `news/` |
| `report-accessibility` | Audits the project for WCAG 2.2 accessibility issues and generates an HTML report |
| `squad` | Runs the full agent squad pipeline for a given GitHub issue |
| `test-runner` | Detects and runs the project test suite (Jest, Vitest, Playwright, Cypress) |

## Architecture Overview

### Frontend Application

The application follows a **component-driven architecture** with unidirectional data flow:

```
BoardingPage (page)
└── KanbanBoard (layout + DnD context)
    ├── Header (search, user menu)
    ├── Sidebar (navigation)
    ├── FilterTabs (tab-based filtering)
    ├── Column x N (swim lanes: To Do, In Progress, Done, ...)
    │   └── Card x N (draggable task cards)
    └── Modals (Add / Edit / Delete)
```

**State management** uses Zustand, a lightweight store that holds all board data (columns, cards, filters). Components subscribe to only the slices of state they need.

**Drag and drop** is implemented with `@dnd-kit`, providing accessible, keyboard-navigable drag interactions between and within columns.

### AI Agent Platform

The FusionCode platform integrates with this repository through two MCP servers configured in `.fusion-plugin/.mcp.json`:

- **Figma MCP** (`mcp-figma`) — reads design files, nodes, and renders previews
- **Jira MCP** (`mcp-atlassian`) — reads and updates project issues on Atlassian Jira

The squad pipeline runs as follows:

```
GitHub Issue
     |
     v
[Figma to Code Agent]  — reads Figma design, generates HTML/CSS or React implementation
     |
     v
[Figma Audit Agent]    — validates implementation against the original Figma design
     |
     v
Commit + Pull Request
```

## Squad Workflow

To run the full agent squad for a GitHub issue, use the `squad` skill from within the FusionCode CLI:

```bash
node cli/bin/fusion.js squad <issue-number> --cwd .
```

Or trigger it via the Claude Code plugin:

```
/squad <issue-number>
```

Individual skills can also be triggered directly, for example:

```
/figma-extract https://www.figma.com/design/<file_key>/<name>?node-id=<node_id>
/deploy-check
/code-review
/report-accessibility
```

## Contributing

1. Create a branch from `main` with a descriptive name
2. Make your changes following the existing code style
3. Use semantic commit messages (`feat:`, `fix:`, `docs:`, `refactor:`, `chore:`)
4. Open a Pull Request with a clear description of the changes
5. Ensure the deploy check passes before requesting review

### Commit message format

```
<type>: <short description>

<optional body>

Co-Authored-By: FusionCode <noreply@brq.com>
```

Types: `feat`, `fix`, `docs`, `refactor`, `test`, `chore`
